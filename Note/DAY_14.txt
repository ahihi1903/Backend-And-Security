🧠 1️⃣ Bản chất của next

Trong server bạn có:

async function next(err) {
  if (err) {
    return errorHandler(err, req, res);
  }

  const handler = handlers[index++];
  ...
}

👉 Đây là function do bạn tự viết

🔥 2️⃣ asyncHandler nhận next từ đâu?
export default function asyncHandler(fn) {
  return async (req, res, next) => {
    try {
      await fn(req, res, next);
    } catch (err) {
      next(err); // 👈 dùng next từ tham số
    }
  };
}

👉 next ở đây không phải import

👉 nó đến từ:

server truyền vào khi gọi handler
🔄 3️⃣ Flow thật sự
Trong server:

👉 bạn truyền next vào đây

Handler là gì?

👉 tức là:

(req, res, next) được truyền vào asyncHandler

🧠 5️⃣ Đây là gì trong JS?

👉 Đây là:

Function parameter passing
Ví dụ đơn giản
function A(fn) {
  fn("hello");
}

function B(msg) {
  console.log(msg);
}

A(B); // 👉 B được truyền vào A
Tương tự backend của bạn
server → truyền next
asyncHandler → nhận next
→ gọi next(err)
🔥 6️⃣ So sánh với Express

Express.js làm y hệt:

app.use((req, res, next) => {
  next(err);
});

👉 next cũng do framework truyền vào
